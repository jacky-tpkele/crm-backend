## 🥇 第 1 梯队 — 高价值、必做（安全 + 体验跨越）

### T1.1 JWT_SECRET / 用户密码强制启动校验  🟩
**问题**：[api/index.js:14-16](api/index.js#L14) JWT_SECRET 有默认值 `'xhon-crm-secret-2025'`，CRM_USERNAME/PASSWORD 有默认值 `TPKELE / 662255`。Vercel 环境变量没设的话会用这些默认值 → 任何人可以伪造 token 或猜密码。
**做法**：
- 启动时检查 `process.env.JWT_SECRET` 必须存在且长度 ≥ 32
- `CRM_USERNAME` 和 `CRM_PASSWORD` 必须存在
- 任一缺失 → 进程抛出明确错误退出，不再回退默认值
**预估**：5 分钟改 + 5 分钟核对 Vercel 环境变量
**风险**：如果 Vercel 上没正确配置变量，部署会失败 → 反而帮你发现配置缺失

### T1.2 全局搜索 Ctrl+K  🟩
**问题**：找一张订单要先点订单页 → 输关键字。找客户、产品同理。每个动作至少 3 次点击。
**做法**：
- 顶栏加搜索框 + 全局快捷键 Ctrl+K / Cmd+K 拉起
- 索引来源：customers / products / orders / suppliers / logistics
- 回车跳到对应详情或列表筛选
- 模糊匹配编号、中文名、英文名、单号
**预估**：半天

### T1.3 仪表盘 KPI 钻取 + 订单全流程进度条  🟩
**问题**：
1. 仪表盘的数字（销售总额、利润、订单数等）**不能点** — 看到异常没法直接钻进去看明细
2. 看一张订单当前停在哪一步（询盘 → 报价 → PI → 订单 → 采购单 → 物流 → 收款）没有聚合视图

**做法**：
- 每个 KPI 卡片加 `onclick` 跳到对应列表 + 预设过滤器（如点"本月销售"→ 跳订单页 + 自动筛本月）
- 仪表盘新加一个"订单流水线"区块：每张订单一行胶囊进度条，emerald 已完成、灰色未完成
- 数据来源：聚合 inquiries / orders / purchase_orders / logistics 状态

**预估**：1 天

## 🥈 第 2 梯队 — 性能 / 可维护性（数据量起来再做）

### T2.1 列表分页 + 服务端搜索  🟩
**问题**：`/api/orders`、`/api/products`、`/api/customers` 一次返回全量。数据上千后明显卡。
**做法**：所有列表接口加 `?page=N&limit=50&q=keyword`，前端改成分页器。
**预估**：1 天

### T2.2 产品图片迁 Supabase Storage  ⬜
**问题**：现在产品图片是 base64 存数据库 TEXT 列（[dashboard.html](dashboard.html) 上次改的）。短期 OK，超 50 个产品后 `/api/products` 响应会到 MB 级。
**做法**：建 `product-images` bucket，加 `/api/upload/product-image` 接口，前端改成传 URL；写一次性脚本把已存 base64 推到 Storage 换 URL。
**预估**：半天 + Supabase 配置 15 分钟
**触发条件**：产品数 > 30 后做

### T2.3 清理 3 套创建订单 + 5 条遗留路由  🟩
**问题**：`POST /api/orders` + `POST /api/save-order` + `POST /api/orders/v2` 三套并存。`/suppliers`、`/orders-full`、`/order-detail/*`、`/orders/*`、`/delete-order/*` 5 条无前缀路由也还在 vercel.json 里 rewrite。
**做法**：保留 `/api/orders/v2`（最新）+ 标准 RESTful `/api/*`，删除其他；同步清 vercel.json。
**预估**：半天 + 仔细测一遍

### T2.4 dashboard.html 拆分  ⬜
**问题**：188KB 单文件，首屏要全量解析。
**做法**：考虑按页面拆成独立 HTML（像 logistics/email 已经独立），或者按需懒加载页面内容。
**预估**：1~2 天，工程量较大
**触发条件**：用户开始抱怨打开慢再做
